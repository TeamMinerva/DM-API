# Checklist Geral de DoR (Definition of Ready)
Um item só pode entrar em uma Sprint se atender aos seguintes critérios:
- A User Story está claramente descrita e compreensível para todos.
- Os critérios de aceitação estão definidos.
- O impacto e a prioridade foram definidos.
- Não existem dependências externas bloqueando o desenvolvimento.
- O esforço foi estimado (ex.: Story Points/Horas).

# Checklist Geral de DoD (Definition of Done)
Uma User Story só pode ser considerada concluída se atender aos seguintes critérios:
- Código implementado conforme critérios de aceitação.
- Código revisado.
- Testes no Colab executados e aprovados.
- Testes manuais de funcionalidade aprovados.
- Documentação (quando necessária) atualizada.
- Nenhum bug crítico pendente relacionado à entrega.
- Merge realizado na main sem conflitos.
